CSS Comments

CSS allows for comments, which are essential for documenting your stylesheets, explaining complex rules, or temporarily disabling parts of your code. CSS comments are very similar to C/C++ comments, surrounded by /* and */.

Basic Comment Syntax

Single-Line Comments

Comments can be placed on a single line. Everything between /* and */ is ignored by the browser. 

/* This is a CSS comment */
.single-line-comment {
    color: #e63946; /* This is a CSS comment for the color property */
    font-weight: bold;
}
This text is styled by a rule with a single-line comment.

Multi-Line Comments

Comments can span multiple lines. The comment starts with /* and ends with */, regardless of how many lines are in between. 

/* This is a CSS comment, and it
   can be several lines long without
   any problem whatsoever.
*/
.multi-line-comment {
    background-color: #a8dadc;
    padding: 10px;
    border-radius: 5px;
}
This box is styled by a rule with a multi-line comment.

Important Restrictions: No Nested Comments

It’s crucial to remember that CSS comments cannot be nested. A comment block ends at the first */ it encounters. If you try to nest comments, it will break your stylesheet.

Incorrect Example: Nested Comments

This example demonstrates what happens when you try to nest comments. The "outside" comment will terminate prematurely at the closing of the "inside" comment, causing the rest of the intended comment block to be treated as active CSS rules (or syntax errors). 

/* This is a comment, in which we find
   another comment, which is WRONG
   /* Another comment */ <-- This closes the outer comment prematurely!
   and back to the first comment, which is not a comment.*/
.nested-comment-issue {
    color: red; /* This line is also commented out by the broken block */
}
WARNING: The CSS rule for .nested-comment-issue is intended to be commented out in the example code block above, but due to the incorrect nested comment syntax, only the portion up to the first */ is actually commented. This means the .nested-comment-issue rule will likely not be applied as intended, or may cause parsing errors. (This specific example class is not applied to any element on this page to avoid visual confusion).

One common way to accidentally create "nested" comments is to temporarily comment out a large block of a stylesheet that already contains a comment. Since CSS doesn’t permit nested comments, the "outside" comment will end where the "inside" comment ends.

Comments on the Same Line as Styles

Unlike some other languages (e.g., JavaScript's // or Python's #), CSS does not have a "rest of the line" comment pattern. The only comment pattern is /* */. Therefore, if you wish to place comments on the same line as markup, you need to be careful about how you place them, ensuring each line or segment of comment is properly wrapped.

Correct Way to Comment on Multiple Lines Alongside Styles

Each line of the comment must be explicitly wrapped in /* and */. 

.correct-same-line {
    color: #457b9d; /* This CSS comment is on the same line */
    background-color: #f1faee; /* as the declaration, and each line */
    padding: 8px; /* needs its own comment markers. */
    border: 1px solid #1d3557; /* This ensures all styles are applied. */
}
This box is styled correctly, with comments alongside each property.

Incorrect Way to Comment on Multiple Lines Alongside Styles

If each line isn't marked off, most of the stylesheet will become part of the comment and thus will not work. 

.incorrect-same-line {
    color: #a7c957; /* This CSS comment is several lines
    background-color: #f2e8cf; /* long, but since it is not wrapped
    padding: 8px; /* in comment markers, the last three
    border: 1px solid #283618; /* styles are part of the comment. */
}

In the example above, only the color property will be applied to .incorrect-same-line. The rest of the rules (background-color, padding, border), being part of the unclosed comment, are ignored by the browser’s rendering engine.

This box should have a light yellow background, padding, and a dark border, but only its text color is applied due to the broken comment in the CSS.
NOTE: CSS comments are treated by the CSS parser as if they do not exist at all, and so do not count as whitespace for parsing purposes. This means you can even put them into the middle of rules—even right inside declarations!

For example, this is valid CSS, though not commonly used for readability: 

.comment-inside-declaration {
    font-size: /* This comment is inside the value */ 20px;
    line-height: 1.5;
    text-decoration: underline /* This comment is after the value */;
    border: 2px solid /* Comment before value */ #e76f51;
}
This text demonstrates comments placed inside CSS declarations.
